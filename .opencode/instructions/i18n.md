# i18n Instructions

## Zakres

Stosuj te instrukcje dla plików związanych z tłumaczeniami, językami, kluczami i treściami user-facing.

## Zasady

- Domyślną biblioteką tłumaczeń jest `i18next` + `react-i18next`.
- Wszystkie stringi widoczne dla użytkownika trzymaj wyłącznie w plikach JSON w `src/i18n/messages/`.
- Nie hardcoduj tekstów UI w komponentach.
- Klucze tłumaczeń mają być czytelne, przewidywalne i konsekwentne.
- Klucze pisz po angielsku, wartości w języku docelowym.
- Struktura kluczy musi być identyczna we wszystkich plikach językowych.
- Używaj maksymalnie dwóch poziomów zagnieżdżenia.
- Namespace odpowiada sekcji strony.
- Fallback językowy ma być zdefiniowany jasno w konfiguracji i18n.
- Nie wprowadzaj alternatywnych bibliotek tłumaczeń bez jawnej zgody użytkownika.

## Struktura plików

```text
src/i18n/
  index.ts
  messages/
    pl.json
    en.json
```

## Workflow przy dodawaniu nowej sekcji

1. Dodaj namespace sekcji do wszystkich plików językowych jednocześnie.
2. Wypełnij wszystkie klucze w każdym języku.
3. Użyj hooka tłumaczeń w komponencie sekcji.

## Namespace'y dla aplikacji workflow

- Dla aplikacji z wieloma route'ami i auth grupuj klucze według obszaru produktu, np. `auth`, `mobile`, `admin`, `dashboard`, oprócz globalnych `common`, `nav`, `footer` i `brand`.
- Dla treści wspólnych dla kilku ekranów preferuj małe, przewidywalne namespace'y zamiast wrzucania wszystkiego do `common`.
- Jeśli backend zwraca status domenowy albo komunikaty techniczne, nie traktuj ich jako źródła copy UI. Widoczne dla użytkownika teksty, nagłówki, CTA, helper text i loading state nadal żyją w `src/i18n/messages/`.

## Obowiązkowe klucze — każdy projekt

```json
{
  "brand": { "name": "[nazwa firmy]" },
  "common": { "sending": "Wysyłanie...", "loading": "Ładowanie..." },
  "nav": {
    "logoAriaLabel": "[nazwa firmy] — strona główna",
    "menuAriaLabel": "Menu"
  },
  "footer": {
    "navAriaLabel": "Nawigacja stopki",
    "legal": { "privacy": "Polityka prywatności", "terms": "Regulamin" }
  }
}
```

## Czego nigdy nie robisz

- Nie wpisujesz nazwy marki bezpośrednio w JSX. Zawsze używaj tłumaczenia.
- Nie hardcodujesz monogramu, skrótu ani tekstowego logo marki w JSX.
- Nie wpisujesz `aria-label` jako stringa.
- Nie wpisujesz stanów UI i komunikatów formularzy jako stringów.
- Nie zostawiasz dead keys w plikach JSON.
- Nie konkatujesz tłumaczeń, jeśli można użyć interpolacji.
